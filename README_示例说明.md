# 当前项目示例说明

这些文件只用于让其他大模型参考交付物格式和审计粒度，不代表未来产品的固定筛选标准。

## 可以参考什么

- Excel的sheet结构。
- 筛选总表字段。
- 纳入、排除、待确认的记录方式。
- 适应性评估表的组织方式。
- 安全性能数据摘录表的组织方式。
- Word报告章节结构。
- 无LibreOffice情况下，应如何说明视觉QA限制。

## 不能照搬什么

- 不能照搬UAS纳排规则。
- 不能照搬一次性输尿管镜纳排规则。
- 不能照搬D/A/P/R/T/O/S/C具体结论。
- 不能照搬分层数量或统计结果。
- 不能把“示例中纳入的研究类型”当成所有产品都必须纳入。

未来换产品时，必须先填写产品变量表，再重新定义关键词、纳入标准、排除标准、分层和适应性评估规则。没有LibreOffice时，不能声称完成了DOCX页面视觉QA。
